# .CLAUDE.md — ezsftp
# Last updated: 2026-03-11 | Evidence: ezsftp.sh (full read, 264 lines)

## What It Is
Single Bash script v1.6 (264 lines) by Jamal Al-Sarraf.
Provisions/revokes chrooted SFTP-only users. Has both interactive menu and
non-interactive CLI mode. Requires root.

## What It Manages Per User
- Linux user + group (`sftpaccess` by default)
- Chroot jail under `${CHROOT_BASE:-/chroot}/<username>`
- Bind mount entry in `/etc/fstab` (shared dir)
- SSH key pair (OpenSSH + PuTTY formats) → `${KEY_DIR:-/root/created_keys}`
- Per-user SSHD config fragment under `${SSHD_FRAG_DIR:-/etc/ssh/sshd_config.d}/`
- SSHD reload after config changes

## Interactive Menu (default mode)
```
1   Add user    (live or dry depending on mode)
2   Remove user
3   Toggle DRY-RUN (switch LIVE ↔ DRY)
4   Show help
Q   Quit
```

## Non-Interactive CLI Mode
```bash
bash ezsftp.sh --help
bash ezsftp.sh --dry-run --add testuser           # preview (old-style flag)
bash ezsftp.sh --non-interactive --action add --user <name>
bash ezsftp.sh --non-interactive --action remove --user <name>
sudo bash ezsftp.sh --non-interactive --action add --user <name>    # real run
```

## Configurable Defaults (env vars override)
| Variable | Default | Scope |
|---|---|---|
| `CHROOT_BASE` | `/chroot` | Base for all chroot jails |
| `SHARED_DIR` | `/shared` | Shared directory bind-mounted in |
| `GROUP_NAME` | `sftpaccess` | Managed SFTP group |
| `KEY_DIR` | `/root/created_keys` | Generated SSH key output |
| `FSTAB_FILE` | `/etc/fstab` | fstab target |
| `SSHD_FRAG_DIR` | `/etc/ssh/sshd_config.d` | Per-user SSHD fragment dir |
| `DRY_RUN` | `0` | `DRY_RUN=1` same as `--dry-run` |

## Validation
```bash
shellcheck ezsftp.sh
sudo bash ezsftp.sh --non-interactive --action add --user testuser --dry-run

```

## Safety-Critical Rules
- ALWAYS test with `--dry-run` before real execution.
- `--action remove` is destructive: removes user, chroot, fstab entry. Dry-run first.
- SSHD changes: fragment files only — never edit `/etc/ssh/sshd_config` directly.
- After any real provisioning run: `sshd -t` to validate SSHD config syntax.
- Script is idempotent — safe to re-run on existing users.
- `DRY_RUN` mode must prevent ALL writes: no filesystem, no fstab, no SSHD config.
