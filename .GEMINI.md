# .GEMINI.md - ezsftp

This project is a single, idempotent Bash script, `ezsftp.sh`, for provisioning and revoking chrooted SFTP-only users.

### Functionality
The script automates all necessary steps for creating a secure, isolated SFTP environment for a user:
- User and group creation (`useradd`, `groupadd`).
- Chroot directory setup (default: `/chroot/<user>`).
- FSTAB bind mounts for home and shared directories.
- SSH key generation (OpenSSH and PuTTY `.ppk` formats).
- Per-user SSH daemon configuration fragments in `/etc/ssh/sshd_config.d/`.

### Execution Modes
The script must be run with `sudo`.

- **Interactive Menu:**
  ```bash
  sudo ./ezsftp.sh
  ```
  This provides a menu to add users, remove users, or toggle dry-run mode.

- **Non-Interactive (for automation):**
  ```bash
  # To add a user
  sudo ./ezsftp.sh --non-interactive --action add --user <username>

  # To remove a user
  sudo ./ezsftp.sh --non-interactive --action remove --user <username>
  ```

### Safety Features
- **Idempotent:** Safe to run multiple times without causing errors or duplicate entries.
- **DRY-RUN Mode:** Preview all changes without making any modifications to the system.
  ```bash
  # Start in dry-run mode
  sudo ./ezsftp.sh --dry-run

  # Or use an environment variable
  DRY_RUN=1 sudo ./ezsftp.sh
  ```

### Configuration
The script's behavior can be customized with environment variables. For example:
- `CHROOT_BASE`: Change the base directory for chroot jails.
- `SHARED_DIR`: Specify a different shared directory.
- `KEY_DIR`: Change where generated SSH keys are stored.

### Validation
- `shellcheck ezsftp.sh`: Check for common scripting errors.
- `bash -n ezsftp.sh`: Validate the script's syntax without executing it.
